1. 🤖 About the bot

@anti_spam_ua_bot is a powerful tool for automatic moderation of Telegram groups. It analyzes all messages in real time, deletes prohibited content and applies punishments for rule violations.

Main features:

  • Deleting profanity, insults, suspicious phrases (ads, phishing).
  • Link blocking (configurable).
  • Anti‑flood (limits on messages per time).
  • Duplicate message detection (identical messages within 24h).
  • Captcha for new members (in development).
  • Flexible punishment system (see section 4).
  • Convenient control panel for admins.
  • Password protection for private chats (optional).

The bot is written in Python (aiogram 2.25.1) and uses SQLite to store settings.

2. ⚡ Quick start

2.1 Adding the bot to a group

  1. Open Group Settings → Administrators.
  2. Click Add administrator and find @anti_spam_ua_bot.
  3. Be sure to grant the following permissions:
    • ✅ Delete messages
    • ✅ Ban users
  4. After adding, enter the command /panel in the group to configure.
💡 Tip: If the bot does not respond to commands, check its status with /diag – it will show whether the bot has administrator rights.

3. 🎛 Control panel (/panel)

The /panel command is the main admin tool. Type it in the group, and the bot will send you a private message with a settings keyboard. If it cannot write to you (you haven't started a dialog), the panel will be sent to the group (and the command itself is deleted).

ButtonDescription
🔤 Word filterEnable/disable checking of profanity, insults, forbidden words. Uses the list from blocklist.txt.
🔗 Link filterEnable/disable blocking of any URLs. @mentions are NOT blocked!
🌊 Anti‑floodProtection against too many messages in a short time. Default: 4 messages in 5 seconds considered flood.
🤖 CaptchaEnable/disable new member verification (button click). Currently in test mode.
📋 WhitelistIf enabled, links containing whitelisted strings (domains, phrases) are not blocked. The whitelist is edited via the database.
📊 StatusShows current chat settings, number of words in blocklist, warning limits, flood limits, etc.
🔄 Reload blocklistForcibly reread the blocklist.txt file and add new words to the database.

When you press a button, the settings change instantly and you receive a confirmation.

4. ⚖️ Punishment system (updated 18.02.2026)

The bot automatically punishes offenders depending on the type of violation. All punishments are accompanied by message deletion and a chat notification (the notification is deleted after 30 seconds).

Violation typeAction
Profanity, insults, suspicious content🟡 Deletion + warning. After 3 warnings – 🔴 ban for 1 day.
Links to channels / crypto scams (t.me/..., crypto keywords)🟡 Deletion + warning. After 3 warnings – 🔴 ban for 10 days.
Regular links (other URLs)🟡 Deletion + warning. After 3 warnings – 🔴 ban for 1 day.
Flood (≥4 messages in 5 s)🟡 Deletion + warning. After 3 warnings – 🔴 ban for 1 day.
Duplicates (3 identical messages within 24 hours)🔴 Instant ban for 7 days (no warnings).
@mentions✅ NOT BLOCKED.
⚠️ Important: The ban applies only to real users (not to anonymous admins who post on behalf of a channel). If a message is sent on behalf of a channel, it is only deleted without punishment.

After each punishment, the bot sends an ad message to the chat with a 30% probability (e.g., about transportation services). It is automatically deleted after 2 minutes.

6. 🔧 Advanced features

6.1 Whitelist

If whitelist_enabled is on, links containing strings from the whitelist are not blocked. The whitelist is stored in the whitelist table. You can add entries via SQL or through code (temporarily).

6.2 Blocklist (forbidden words)

The list of forbidden words is stored in the blocklist.txt file (one word per line) and in the database. The bot automatically checks if the file has changed (by modification time) and updates the database. You can update manually with /reload_blocklist or via the panel.

Words are normalized (replace Latin with Cyrillic, remove extra characters) and checked as whole words (with \\b boundaries) to avoid false positives.

6.3 Blocklist exceptions

Some phrases can be added to the bad_exceptions table; then they will not be considered a violation even if they contain forbidden roots.

6.4 Password protection (private chats)

To prevent unauthorized access to the bot in private messages, you can set a password in the .env file (BOT_PASSWORD=your_secret). Users must then type /login your_password in private before using any other commands. This does not affect group chats.

7. 🛠 For developers (how to change the ad text)

The bot has built‑in advertising that is sent with a 30% probability after a punishment, but no more than once every 5 minutes in a chat. To change the text or link, edit the main.py file.

7.1 Where is the ad text located?

In the main.py file, find the variable AD_MESSAGE (around the configuration section, after imports). It looks like this:

AD_MESSAGE = "🚛 Вантажні перевезення по Києву та області: https://t.me/vantazhnik_kyiv"

You can replace this string with any other text, e.g., your own link or message.

7.2 How to change the display probability?

Nearby there is the variable AD_PROBABILITY = 0.3 (30%). Change it to, say, 0.5 for 50% or 1.0 for 100%.

7.3 How to change the cooldown between ads?

The variable AD_COOLDOWN = 300 is 300 seconds (5 minutes). Increase or decrease as needed.

7.4 Project structure

  • main.py – main bot file (all code).
  • blocklist.txt – list of forbidden words.
  • .env – file with bot token (and optional password).
  • bot.db – SQLite database (created automatically).
🔧 To change the ad: find the AD_MESSAGE line and edit it. After changes, restart the bot.

8. ❓ Frequently Asked Questions

Why doesn't the bot respond to commands?

Check if the bot is added as an administrator and has message deletion rights. Run /diag – it will show the bot's status in the chat.

Can I add my own words to the blocklist?

Yes, just add them to the blocklist.txt file (one word per line) and run /reload_blocklist or click the corresponding button in the panel.

Does the bot delete messages from administrators?

Yes, now the bot does not make exceptions for admins. All messages are checked equally. If an admin breaks the rules, they get punished.

Why does the bot send ads?

Ads help support the server and the bot's development. You can change the text or disable it (see section 7).

Can I disable anti‑flood only for certain users?

Currently no. But you can add users to the whitelist – it only applies to links, not flood.

How does the password protection work?

If you set BOT_PASSWORD in .env, anyone who writes to the bot privately must first type /login password to access any commands. This prevents unauthorized use of the bot's private features.
